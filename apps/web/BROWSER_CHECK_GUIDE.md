# 브라우저에서 데이터베이스 연결 확인 가이드

## 현재 상황

브라우저에서 `http://localhost:3000/api/products/brands` 접속 시 빈 화면이 보이는 경우:

### 가능한 원인

1. **정상 작동 (데이터 없음)**
   - API가 빈 배열 `[]`을 반환
   - 데이터베이스에 브랜드 데이터가 없음
   - 이것은 정상입니다!

2. **데이터베이스 연결 실패**
   - 에러 핸들링으로 빈 배열 반환
   - 앱은 계속 작동하지만 데이터는 없음

## 확인 방법

### 방법 1: 브라우저 개발자 도구 (F12)

1. **F12 키 누르기**
2. **Console 탭 확인**
   - 데이터베이스 연결 오류 메시지 확인
   - Prisma 관련 오류 확인
3. **Network 탭 확인**
   - `/api/products/brands` 요청 클릭
   - **Response 탭**에서 실제 응답 확인
   - 빈 배열 `[]`이면 정상 (데이터 없음)
   - 오류 메시지가 있으면 문제

### 방법 2: 응답 형식 확인

브라우저에서:
1. 개발자 도구 열기 (F12)
2. Network 탭 선택
3. `/api/products/brands` 요청 찾기
4. Response 탭 클릭
5. 응답 내용 확인:
   - `[]` → 정상 (데이터 없음)
   - `{"error": "..."}` → 오류 발생

### 방법 3: JSON 형식으로 확인

브라우저 주소창에 입력:
```
http://localhost:3000/api/products/brands
```

**예상 결과:**
- 성공: `[]` (빈 배열)
- 실패: `{"error":"Failed to fetch brands"}`

## 다음 단계

### 데이터베이스에 데이터가 있는지 확인

Supabase 대시보드에서:
1. **Table Editor**로 이동
2. **Product** 테이블 확인
3. 데이터가 있는지 확인

### 데이터가 없다면

정상입니다! 빈 배열 `[]`을 반환하는 것이 맞습니다.

### 데이터가 있는데 빈 배열이 반환된다면

데이터베이스 연결 문제일 수 있습니다:
1. 서버 로그 확인
2. 브라우저 콘솔 오류 확인
3. `.env.local` 파일의 `DATABASE_URL` 확인

## 빠른 확인

브라우저 개발자 도구 (F12) → Network 탭 → `/api/products/brands` → Response 탭

응답이 `[]`이면 정상입니다!
