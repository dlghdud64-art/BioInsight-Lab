# Pilot Scenario: Budget Exception Decision

## User Role

Approver.

## Goal

Review a budget-impacting request and choose proceed, substitute, or exception approval.

## Starting State

There is a procurement request with budget risk or approval impact.

## Steps

1. Open budget or approval surface.
2. Identify the blocked or risky request.
3. Review budget impact.
4. Review AI-supported decision options.
5. Choose a reviewed next action.

## Success Criteria

- Budget is shown as an operational constraint, not a standalone admin list.
- AI presents three decision options.
- Operator review is required before applying any action.
- The next step is approval, substitute review, or hold.

## Failure Signals

- Budget page shows only static financial rows.
- AI recommends one line without operational options.
- Approval or exception can be bypassed.
- The user cannot tell which request is affected.
