## 1. 핵심 판단

| 항목 | 데스크탑 | 모바일 |
|------|---------|--------|
| 검색 → 비교 → 견적 | 검색/비교/견적 요청 화면 존재, 운영 가능 | 견적 조회/상태변경/메모/주문전환 가능, 견적 요청은 데스크탑 전용 |
| 견적 → 구매 전환 | dashboard/quotes에서 관리 | 모바일에서 주문전환 실동작 (PATCH status=PURCHASED) |
| 구매 → 입고 | dashboard/purchases에서 관리 | 간편입력/TSV/CSV 등록 → 재고 반영(lookupInventory → lot-receive) 파이프라인 완성 |
| 재고 Lot 관리 | dashboard/inventory에서 조회 | Lot 입고/출고/위치지정/라벨인쇄/점검 전체 실동작 |
| 안전관리 | dashboard/safety 존재 | SafetyInfoCard + 점검 체크리스트 + 사진첨부 |
| QR 스캔 | dashboard/inventory/scan 존재 | 카메라 스캔 + 수동검색 + 액션시트 + Fallback UI |
| 푸시 알림 | N/A | expo-notifications 딥링크 (견적/구매/재고) |

**결론**: 모바일 앱은 P0 핵심 플로우(견적조회→구매등록→재고입고) 전체 처리 가능 상태. 데스크탑은 검색/비교/견적 요청이 주 역할이며, 모바일은 현장 운영(입출고/점검/스캔)에 특화.

### 2-3. 모바일 전용 처리 화면

- 입고/출고/위치/라벨/점검: 모바일에서만 전용 화면 제공 (데스크탑은 모달/인라인)

- 푸시 알림 딥링크: 모바일 전용
- 사진 첨부: 모바일 카메라/앨범 네이티브
