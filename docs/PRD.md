1. 프로젝트 개요
1.1 목표

바이오·제약 연구/품질/생산 현장에서 사용하는 시약·소모품·장비에 대해,

검색 → 비교 → 견적용 리스트 생성 →
그룹웨어/전자결재용 텍스트·파일 생성 →
(추후) 구매 내역·예산 및 안전·규제 정보까지

를 하나의 웹 서비스(도구) 안에서 자연스럽게 이어주는 구매 준비 플랫폼을 만드는 것이 목표.

포지션: “시약 쇼핑몰”이 아니라,
연구실/조직의 시약·장비 구매 준비/정리 도구 + 인사이트 도구

1.2 문제 정의

현재 연구/품질/생산 현장의 공통 Pain Point:

제품 정보가 벤더별 사이트, PDF 카탈로그, 엑셀, 메일 등으로 분산

연구자/실사용자는

제품명·스펙·가격을 탭 여러 개 열어 비교

엑셀·노션 등에 수동 정리

사내 그룹웨어 양식에 다시 복붙

구매 담당자는

제품명이 뒤섞인 요청 엑셀/메일을 받아, 다시 정리해야 함

안전/규제 측면:

MSDS/SDS, 식약처 정보, 위험·유해 문구, 보관 조건은 또 다른 포털/문서를 뒤져야 함

→ 결과적으로:

시간 소모 크고

실수·누락 위험 높고

조직 관점에서 예산/위험 관리가 어렵다.

1.3 핵심 가치 제안 (Value Proposition)

한 번의 검색/입력 → 여러 벤더 제품 자동 정리

GPT로 의미 분석, 유사 제품 추천, 필터 정리

비교/견적 리스트를 구조화된 형식으로 자동 생성

장바구니처럼 후보를 모으고, 한 번에 보기 좋게 정리

조직에서 실제로 쓰는 그룹웨어/엑셀 양식에 바로 붙여넣기

TSV/테이블 텍스트/CSV 등으로 자동 변환

안전·규제 정보까지 한 번에 확인

제품 상세에서 MSDS/SDS, 식약처/규제 링크, 기본 취급 요약 확인

장기적으로는 예산·구매 내역·안전관리까지 커버하는 플랫폼

2. 타깃 유저 & 페르소나
2.1 주요 사용자 유형

연구자 (R&D Scientist)

신규 실험 설계, 시약/키트 비교, Pilot 구매

요구: 스펙/성능 중심의 비교, 비슷한 타깃/용도의 제품 추천

QC/QA 실무자

시험법에 맞는 시약/Reference, 장비 유지 관리

요구: 동일 규격/대체품 비교, 규격/Grade/제조사 신뢰성

생산/공정 엔지니어

공정용 소모품/장비·부품 구매

요구: 카탈로그 번호, 호환 장비, 납기/안정공급

구매/자재 담당자

연구/생산 측에서 올라온 리스트를 예산/계약 관점에서 검토

요구: 벤더/카테고리/기간별 집계, 예산 대비 사용, 위험물/규제 물질 관리

안전관리자/환경·보건 담당 (P2 이후 중요)

고위험 물질/유해물질 관리, MSDS/SDS 관리, 규제 대응

요구: 품목별 위험군, MSDS/SDS 유무, 보관·취급 주의사항 집계

2.2 사용 환경

기업/연구소 내부 네트워크 환경 (프록시/보안망 가능)

크롬/엣지 최신 버전 기준

사내 그룹웨어(전자결재 시스템)와 직접 연동 대신,

복사/붙여넣기 + 파일 업로드 중심 플로우를 1차 목표로 한다.

3. 주요 유저 플로우
3.1 플로우 A – 검색 → 비교 → 견적 준비

홈/검색 화면에서 키워드 입력 (제품명/타깃/벤더 등)

GPT가 검색어를 분석 → 관련 제품 후보 리스트 표시

사용자가 필요 제품을 선택/추가 → 비교 테이블 구성

비교 테이블에서 최종 후보 선택 → “구매 요청 리스트”에 담기

구매 요청 리스트에서 수량/비고를 정리

그룹웨어용 텍스트/TSV/CSV로 변환 → 그룹웨어 양식에 붙여넣기/업로드

3.2 플로우 B – 프로토콜 텍스트 → 필드 추출 → 제품 추천

사용자가 기존 실험 프로토콜 일부를 텍스트로 붙여넣기

GPT가 프로토콜 텍스트를 분석:

실험 유형, 타깃, 농도/부피, 사용 장비 유형 등

추출된 필드를 기반으로 검색 필터 자동 설정

관련 제품 리스트/비교 테이블 생성 → 이후 흐름은 플로우 A와 동일

Mode B = 텍스트 붙여넣기 중심 (P0)
Mode A = PDF 업로드 후 자동 파싱 (P1 이후)

3.3 플로우 C – 구매 내역 / 예산 확인 (P1)

사용자가 과거 구매 내역을 CSV/엑셀로 업로드
혹은 이 서비스에서 만든 리스트에 “구매 완료” 태그 부여

기간/프로젝트/벤더/카테고리별로 필터링

예산 사용률/비중을 간단한 그래프/테이블로 조회

3.4 플로우 D – 안전·규제 정보 확인 (P1~P2)

제품 상세 페이지에서 “안전 · 규제 정보” 섹션 확인

MSDS/SDS 링크, 식약처/규제 포털 링크, 기본 취급 요약

(P2) 안전관리자용 뷰에서

고위험 물질, MSDS 없는 품목, 규제 코드 포함 품목만 필터링/리스트 확인

6.1 도입 배경

시약/화학물질/장비 사용 시 필수적으로 확인해야 하는 정보:

연구자/실사용자 + QC/안전관리자/구매 담당자 모두에게 가치 ↑

단순 검색/비교를 넘어 조직의 구매·안전 관리 도구로 격상

6.2 요구사항 (P1 – MVP 확장 기능)
6.2.1 제품 상세 내 “안전 · 규제 정보” 섹션

위치

제품 상세 카드 내 탭 혹은 하단 섹션:

[기본 정보] / [비교 정보] / [안전·규제 정보]

구성 요소

MSDS / SDS 보기 버튼

Product 필드: msds_url: string | null

동작:

URL이 있으면 새 탭으로 PDF/웹페이지 열기

없으면 “등록된 SDS가 없습니다” 안내

국내 규제 정보 바로가기 버튼

공통 링크 목록:

식약처 의약품/시약 정보/포털

MSDS/SDS 통합 검색 서비스

화학물질관리법/산업안전보건법 관련 공식 페이지

초기에는 제품별 매핑 없이, 공통 규제 페이지 링크 제공

주의사항 요약 텍스트 (옵션)

Product 필드: safety_note: string | null

예시:

“피부 자극 가능성, 장갑·보안경 착용 권장”

“2~8°C 냉장 보관, 밀폐 용기 사용, 직사광선 피할 것”

내용은 수동 입력 or 추후 AI 추출

UI 예시

[ ⚠ 안전 · 규제 정보 ]

[ MSDS / SDS 보기 ]
[ 국내 규제 정보 페이지 열기 ]

- 사용 전, 취급·보관·폐기 정보를 꼭 확인하세요.
- 예: 2~8°C 냉장 보관, 장갑·보안경 착용 권장

6.2.2 데이터 모델 (P1 범위)

Product 스키마에 아래 필드 추가:

msds_url?: string | null;    // MSDS/SDS PDF 또는 외부 링크
safety_note?: string | null; // 한두 줄 취급 주의사항 요약


규제 사이트 링크:

코드/설정 파일에 정적 정의

예: SAFETY_LINKS = [{ id, name, url }]

6.3 고도화 아이디어 (P2 이후)
6.3.1 안전 필드 구조화

Product에 확장 필드 도입:

hazard_codes?: string[];        // 예: ["H314", "H290"]

UI:

보관: 2~8°C 냉장, 밀폐 용기

6.3.2 안전관리자용 뷰

기능:

조직/팀의 사용 제품 중:
