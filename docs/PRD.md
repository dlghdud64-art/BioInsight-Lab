A. PRD 필수 구성 점검

 목적/범위(P0/P1/P2)가 명확히 분리되어 있는가?

 타깃 유저(구매자 + 벤더)가 페르소나/니즈로 정의되어 있는가?

 핵심 플로우(검색→비교→리스트→내보내기)가 “완주 가능한 형태”로 서술됐는가?

 RFQ/회신이 “워크플로우(상태/엔티티)”로 정의되어 있는가?

 안전·규제는 원문 우선/고지 템플릿이 포함되어 있는가?

 데이터시트 번역/요약은 저장 정책/출처/고지가 포함되어 있는가?

 벤더 데이터 파이프라인(업로드→검증→승인→Publish)이 정의되어 있는가?

 예산 기능이 “최적화”보다 “현황/경고” 중심으로 단계화되어 있는가?

 기술스택/라이브러리가 MVP 범위에 맞게 최소화되어 있는가?

 KPI/이벤트/검증 계획이 실행 가능한 수준인가?

 리스크/법무/컴플라이언스가 P0에서 바로 적용 가능한 원칙으로 작성됐는가?

B. 화면/정보구조(IA) 점검

 Home 데모 Step이 3단으로 통일되어 있는가?

 검색/비교/리스트 화면에서 CTA가 “다음 단계 1개”로 명확한가?

 모바일에서 버튼이 숫자만 보이지 않고 라벨이 노출되는가?

 비교/리스트에 삭제/선택삭제/총액 등 기본 조작이 일관적인가?
