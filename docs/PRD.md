AI 기반 바이오·제약 시약/장비 검색 · 비교 · 품목 리스트 · 구매요청 지원 도구

1. 프로젝트 개요
1.1 서비스 가칭

BioInsight Lab

1.2 목표

바이오·제약 연구, QC, 생산 현장에서 사용하는 시약·소모품·장비를
한 화면에서 검색 → 비교 → 품목 리스트(견적/구매신청용) 생성 → 구매요청까지 지원하는 웹 도구.

사용자는 이 리스트를 사내 그룹웨어/전자결재 시스템에 그대로 붙여넣어
기존 구매 프로세스를 유지하면서, 준비 과정만 자동화.

GPT를 활용해:

검색어 의도/타깃/카테고리 분석

영문 설명/데이터시트 텍스트 요약/번역

최소한의 대체 후보(Alternative) 정보를 보조적으로 제시

구매내역/예산 리포트 기능으로
기간·프로젝트·벤더별 총 구매 금액을 확인하고, 예산 활용을 가늠.

중장기적으로는 팀/조직 단위로 사용하는 B2B 구독형 SaaS를 목표로 하며,
베타 단계에서는 무료/파일럿 형태로 도입 후, 정식 요금제로 전환.

3. 문제 정의

제품/벤더 정보가 여러 사이트에 흩어져 있어 가격·스펙·납기를 한 번에 비교하기 어렵다.

영문 제품 설명/데이터시트가 많아,
읽고 해석·비교하는 데 시간이 많이 든다.

QC/생산은 동일 Grade/규격 유지가 중요한데,
대체품 검토·기록이 번거롭다.

실제 견적/발주는 사내 그룹웨어에서 진행되지만,
그 전에 “구매신청용 품목 리스트”를 만들고 정리하는 과정이 비효율적.

구매가 반복되지만,
**“올해 이 과제/팀/벤더별로 얼마 썼는지”**를 즉시 보기가 어렵다 → 예산 관리/결정이 감에 의존.

4. 메인 유저 플로우
4.1 검색 기반 올인원 플로우 (P0)

사용자(연구자/QC/생산/구매)가 /search 진입

상단 검색창에 키워드 입력

예: Human IL-6 ELISA kit, 0.22um filter, HPLC C18 column

GPT 의도 분석

타깃(Analyte), 카테고리, 실험 유형 분석 → 태그/배지로 표시

검색 결과 리스트에서 제품 후보 확인

제품 행 클릭 → 제품 상세 패널 오픈

리스트/상세에서 제품을

“비교 대상”으로 선택

“품목 리스트(구매요청용)”에 추가

하단/우측의 비교 테이블 + 품목 리스트 테이블에 즉시 반영

수량/비고 입력 후 “구매 요청서 준비”로 이동

/quote 화면에서 품목 리스트 최종 검토 및 내보내기

/quote/request 화면에서
견적 요청 제목/내용/납기/주소 작성 →
요청 내용/품목 요약 모달에서 최종 확인 →
그룹웨어/구매 담당자에게 발송

벤더 응답/견적 비교/발주까지의 “Program 단계”는 P2 이후.

4.2 예산/구매내역 리포트 플로우 (P1)

구매담당자/팀장이 /reports 진입

기간, 조직/팀, 프로젝트, 벤더/카테고리 필터 선택

BioInsight Lab에서 생성된 품목 리스트 및 (옵션) 실제 구매 CSV 기반으로:

총 금액

벤더/카테고리별 합계

월별/분기별 추이

예산 활용/다음 분기 예산 책정 참고

벤더

카테고리

Grade/규격

카탈로그 번호

표시 순서:

좌측: 제품명, 벤더

중간: 카테고리, Grade/규격, 스펙 요약

우측: 가격, 재고/납기

5.2.2 재고/납기 표현

데이터가 있을 경우:

텍스트 + 상태 배지로 표현

1~2일 → 빠른 출고 (초록 계열)

3~7일 → 보통

7일 이상 → 지연 가능

없을 경우:

재고 확인 문구로 통일 또는 컬럼 숨김

5.2.3 스펙 요약

예: Human · ELISA · 96-well

1줄만 보여주고, 길면 … 처리

5.2.4 인터랙션

행 전체 클릭 → 제품 상세 패널 열기

각 행 우측 퀵 액션:

비교 토글

품목 추가 버튼 (품목 리스트에 추가)

가격/재고 없음:

가격 문의, 재고 확인 등 일관된 문구 사용
