# Ontology Decision: 구매 단계 화면의 주체 재정의 (요청자 중심)

- **Status:** 🧭 결정 문서 (제품 경계 결정 — 개별 SPEC의 상위)
- **Last Updated:** 2026-06-02
- **성격:** UI 패치 아님 — **제품 경계(누가 주체인가) 결정.** 구매 운영·발주 관리·구매 리포트 세 화면의 존재 이유 재정의. 이후 §11.35x SPEC은 이 결정을 따른다.

## 0. 깨진 전제
당초: 견적 관리(요청자) → 구매 운영(구매자) → 발주 관리(PO 추적).
현실: 실사용자=요청자(연구원). 구매 담당자는 LabAxis 사용 의무 없음(ERP/전자결재가 일터). → "구매자가 발주 결정하는 화면"인데 그 구매자가 시스템에 없음 = 빈 게이트(dead surface 위험).

## 1. 구매자 유입 평가 → 단기 낮음 → B 확정
- 구매자는 ERP/전자결재에 묶임, KPI(단가·정산·예산·세금계산서)가 LabAxis 밖, needs 불일치.
- 구매 리포트는 유입의 결과지 원인 아님(내부 발주만 집계=반쪽). 순서 거꾸로.
- 구매자 유입 유일 조건 = ERP보다 명백히 편할 때(ERP 연동 전제, 무거움). **편의가 의무를 이긴다. 강제 불가.**

## 2. 채택: B — 요청자 중심 폐루프
LabAxis는 발주 "직전"까지가 실효 영역. 발주 실행은 시스템 밖(구매자/ERP). LabAxis는 발주를 *실행*이 아니라 **의뢰·추적**. 모든 구매 단계 화면 주체 = 요청자/연구소 관리자.

## 3. 세 화면 재정의
| 화면 | 재정의 | 주체 |
| :-- | :-- | :-- |
| 견적 관리 | 유지(견적 요청·회신 수집) | 요청자 |
| 구매 운영 | 발주 직전까지 정리 → 구매자 인계(핸드오프) + 인계 후 상태 추적 | 요청자 |
| 발주 관리 | 외부 발주 상태 추적(외부 발주됨/입고됨 수동 마킹) | 요청자 |
| 구매 리포트 | 요청자·관리자용 "우리 연구소가 뭘·얼마나·어디서 사는지" | 요청자/관리자 |

- 3-1 구매 운영 = **발주 인계 게이트**: PO 자동 발행 ❌ → 발주 요청 패키지화 + 외부 공유 형태 + "외부 발주됨" 수동 마킹. (우측 레일 dead-end 재해석)
- 3-2 발주 관리 = **외부 발주 상태 추적**: 수동 마킹 + (장래) 회신/입고 신호.
- 3-3 구매 리포트 = **관리자 가치**: 요청 데이터만으로 성립(구매자 데이터 불필요).

## 4. 미래의 다리 (강제 아닌 편의)
- 발주 인계 산출물을 ERP/전자결재가 받을 구조화 export 로. 나중 구매자 진입점. (§11.348 연결) 강제 X, 편의로만 유인.

## 5. 핵심 원칙 (이후 SPEC 공통)
- 모든 구매 단계 화면 주체 = 요청자/관리자. 구매자 부재 전제.
- LabAxis = 발주 의뢰·추적, 실행은 외부(안 쓰는 단계 생성 금지).
- dead surface 금지(안 쓰는 주체용 화면 정교화 금지).
- 구매 리포트는 관리자 가치로(내부 데이터만으로 의미 있는 범위).
- canonical(견적·발주 상태)=서버, 외부 발주 상태=수동 마킹이 truth.

## 6. 후속 SPEC (이 결정을 따름, 미착수 백로그)
- §11.352 구매 운영 재정의(발주 인계 게이트) — 우측 레일 dead-end 해소. **백로그 상위(라이브 결함).**
- §11.353 발주 관리 = 외부 발주 상태 추적(수동 마킹 + 입고 인계).
- §11.354 구매 리포트 = 관리자 관점 재정의(지출·공급사·품목). ← §11.350(지출 분석) 과 동일 surface, 같은 철학.
- (연결) §11.348 발주→회신 폐루프 / §11.346 재고 정책.

## 7. 우선순위
경계 확정 문서. 후속 §11.352~354 = 백로그(대기열 §11.335b·§11.336·§11.349·§11.351 + vitest 뒤). §11.352(구매 운영 dead-end)는 백로그 내 상위.

## 8. 전제/미해결
- ✅ 요청자=실사용자, 구매자=의무 없음 / ✅ 구매자 단기 유입 낮음 → B.
- 🔎 발주 관리 PO 상태 수동 마킹 vs 장래 신호 연결 — §11.353.
- 🔎 ERP/전자결재 export 포맷 — 미래 다리, 미정.

## 9. Notes
- 핵심 = "기능 추가" 아니라 "경계 축소" — 안 쓰는 주체 정교화 멈추고 쓰는 주체(요청자/관리자) 집중.
- 구매 리포트 유입미끼 가정 폐기 → 관리자 가치 전환.
