## 0. 한 줄 정의
프로토콜(문서) 업로드 → 시약·품목이 BOM으로 자동 추출 → 같은 다운스트림(검색·비교·견적)으로 흐름. **검색 = 손으로 찾기 / BOM = 프로토콜로 뽑기**, 둘 다 **소싱의 입구**. ERP가 못 주는 요청자 차별점.

---

## 1. 잠긴 결정
- **소싱 진입 = 방향 맞음.** BOM은 소싱의 대안 입구(BOM 화면 Step 4가 "검색·비교·견적"인 게 증거). 올바른 형태 = 소싱 상단 **"검색 / 프로토콜 업로드" 병렬 진입**.
- **MVP 경계 (호영님 직관 = 정확):**
  - ✅ **시약·조성 중심 문서**(시약 프로토콜, 배지·완충액 조성, 시약 리스트) = high-signal. 시약명·CAS → 제품 매핑 직접적. **PoC 대상.**
  - 🚧 **전체 실험 프로토콜**(prose/SOP) = noisy. 시약이 본문에 흩어지고("PBS로 3회 세척"), 암묵 품목·소모품·장비 섞임. 등급·공급사 추천은 텍스트가 아니라 연구자 도메인 판단. → **연구자 인터뷰 게이트 통과 전 MVP 밖.**
- **"제품 추천" 정의 (못 박음):** AI 주관 권유("이거 사세요") 금지. = **추출 BOM 항목 → 카탈로그 매핑**(시약명/CAS → 제품 후보 리스트), deterministic. 환각·fake 추천 차단.
- **기밀 전제:** 무료 티어는 입력을 모델 학습에 사용 → 고객 프로토콜(연구 IP/기밀, GMP)엔 치명. **제품화 = 학습-비사용 유료 티어/Vertex 전제**(비용 아닌 기밀 사유). 비용 지불 가능 확인됨 → 이 매듭은 닫힘.
- **CAS·카탈로그 번호 환각 = 최대 위험.** 없으면 빈칸, 절대 생성 금지(fake success 안티패턴). BOM 검토(사람 검수) step을 초안→검수로 사용.

## 3. PoC 계획 (유료 OK, 좁은 케이스)
1. **시약·조성 문서**로 추출 정확도 측정(시약명/규격/CAS/수량). ⚠️ 고객 실제 프로토콜은 무료 티어 금지(기밀 학습) — 더미/본인 문서로.
2. 멀티모달(Gemini Flash류) = 다중 페이지 PDF·표 레이아웃 한 번에 처리(100만 토큰), native-text PDF면 OCR 불필요. 스캔본 비중이 변수.
3. 추출 → 카탈로그 매핑(deterministic) → 후보 리스트. 매핑 실패는 빈칸(생성 금지).
4. 판정: 정확도 + 검수 부담 → §2 기준.

## 4. 연구자 인터뷰 질문 (실험 프로토콜 게이트 — 4축)

### 축 A. 추출 정확도 기대치
- 프로토콜 한 건에서 "이 정도는 자동으로 잡혀야 쓴다" 하는 최소 항목은? (시약명만? 규격·수량까지? CAS까지?)
- 자동 추출이 80% 맞고 20%는 직접 채워야 한다면 — 쓸 만한가, 안 쓰는가?
- 빈칸(불확실)으로 두는 것 vs 틀린 값을 채우는 것 — 어느 쪽이 더 거슬리는가? (환각 허용도 확인)

### 축 B. 추천 깊이
- "추출된 항목 → 카탈로그 제품 후보 매핑"까지면 충분한가, 등급·공급사·대체품 추천까지 원하는가?
- 등급/공급사 선택은 시스템이 제안해야 하나, 연구자 본인 판단 영역인가? (deterministic 경계 확정)
- "이 시약은 이 제품으로 사세요"를 AI가 단정하면 신뢰가 오르나 떨어지나?

### 축 C. 프로토콜 포맷·스캔 비중
- 실제 프로토콜이 native-text PDF(워드/논문 export)인가, 스캔본(이미지)이 섞이는가? 대략 비중은?
- 한 실험 = 몇 페이지 / 몇 개 문서? (여러 PDF 합본 여부)
- 사내 표준 양식이 있나, 연구자·랩마다 제각각인가?

### 축 D. 검수 부담 수용도
- 추출 후 "BOM 검토"에서 항목당 몇 초/몇 분까지 보정 가능한가? (손 검색 대비 이득이 남는 임계)
- 검수 없이 자동 신뢰는 절대 불가인가, 고신뢰 항목은 자동 통과 허용인가?
- 이 기능이 실제로 "검색 노가다"를 줄여준다고 느낄 시나리오 1개를 말해달라.
