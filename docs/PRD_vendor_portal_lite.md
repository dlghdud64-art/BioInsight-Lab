BioInsight Lab – Vendor Portal Lite PRD (v0.1)

0. 문서 정보

- 목적: 기존 메인 PRD(`docs/PRD.md`)에서 벤더 관련 내용을 분리해, **벤더 포털 라이트(P1)**를 독립적인 기능 단위로 정의
- 대상:
  - 벤더 조직(영업/영업지원/견적/제품 담당)
  - 내부 PM/기획/FE/BE
  - 파트너/벤더 세일즈 미팅용 자료

1. 제품 개요

1.1 목표

- 벤더가 **제각각인 RFQ 포맷/채널** 때문에 겪는 비효율을 줄이고,
  - 누락·오해 감소
  - 회신 시간 단축
  - 가격/납기/MOQ 정보 관리 체계화
- 동시에, 벤더가 **자신의 성과와 고객 선택 패턴**을 볼 수 있는 최소 지표를 제공하는 “라이트 포털”을 만든다.

1.2 포지션

- 풀기능 CPQ/CRM이 아니라,
  - **표준화된 RFQ 수신/회신 + 카탈로그/가격표 관리 + 최소 지표**를 제공하는 경량 포털
- Buyer 쪽 BioInsight Lab과는
  - **RFQ/제품 데이터 레이어만 공유**하고
  - 조직/권한/화면은 분리된 “벤더 전용 뷰”

1.3 범위 (본 문서)

- P1 “Vendor Portal Lite” 범위만 다룸:
  - 로그인/권한(벤더 도메인 기준)
  - RFQ 리스트/상세 보기
  - RFQ 회신 입력(단가/납기/MOQ/대체품)
  - 카탈로그/가격표 업로드(가능 시), diff/publish 라이트 버전
  - 최소 지표(현황/회신률/평균 회신 시간)
- 향후 “Vendor Pro”(고급 지표/자동화/조직별 계약가 관리)는 개념만 언급

2. 타깃 유저 & 페르소나 (벤더 측)

2.1 주요 유저 유형

- 영업/영업지원(Inside Sales)
  - 이메일/엑셀로 오는 견적 요청 정리, 회신 담당
- 견적 담당(Quotation Desk)
  - 단가/납기/MOQ/로트 등 견적 산출, 내부 승인 후 회신
- 제품/기술 담당(PM/Tech)
  - 특정 제품/라인업의 스펙·데이터시트·카탈로그 관리

2.2 벤더 Pain Point (요약)

- RFQ 형식이 제각각 → 필요한 정보 누락/추가 질문 반복
- 가격/납기/MOQ 정보가 여러 시스템/엑셀에 흩어져 있음
- 회신 이후 “선정 여부/이유”를 못 보고, 어떤 RFQ에서 왜 졌는지 알기 어려움

2.3 벤더에게의 가치 제안 (Why Vendor Uses)

- 표준 RFQ로 수신 → 필수 필드 누락 감소, 재질문 횟수 감소
- 카탈로그/가격표를 Verified 데이터로 관리 → 잘못된 문의/스펙 착오 감소
- 링크/엑셀 기반 빠른 회신 입력 → 회신 시간 단축
- (선택) RFQ 수/회신률/평균 회신 시간/선정 신호 → 어디에 리소스를 더 써야 하는지 판단

3. 주요 유저 플로우 (벤더 관점)

3.1 플로우 H-1: RFQ 수신/회신 (핵심 P1)

- Step 1: 벤더 담당자가 포털에 로그인
- Step 2: 본인 회사로 온 RFQ 리스트를 본다
  - 상태/마감일/요청 수량/요청 제품 요약 표시
- Step 3: 특정 RFQ를 열어 요청 제품별로
  - 단가(Unit Price)
  - 납기(Lead Time)
  - MOQ
  - (선택) 제안 대체품
  을 입력하거나, 엑셀 템플릿으로 업로드
- Step 4: 회신 저장/제출
  - Buyer 쪽에서는 회신 내역이 자동 취합/비교 테이블에 반영

3.2 플로우 H-2: 카탈로그/가격표 업데이트 (라이트, 가능 시)

- Step 1: Vendor Admin/Member가 카탈로그/가격표 파일(xlsx/csv)을 업로드
- Step 2: 시스템이 기본 zod 검증 + 간단한 diff 결과를 보여줌
  - 신규/변경/삭제 건수 정도의 요약
- Step 3: 벤더가 변경 내용을 확인 후 “적용(publish)” 버튼
- Step 4: Buyer 쪽에서 해당 벤더 제품에 “Verified” 배지/업데이트 날짜가 반영

3.3 플로우 H-3: 성과/현황 보기 (선택)

- Step 1: 벤더 담당자가 “대시보드/현황” 메뉴 진입
- Step 2: 기간/제품군 필터로 RFQ 현황을 본다
  - RFQ 수, 회신률, 평균 회신 시간
  - (가능하면) 선정된 RFQ 수/비율, 기본적인 탈락 사유 카테고리
- Step 3: 어떤 라인/고객군에 더 집중해야 할지 내부 논의에 활용

4. 기능 범위 (Vendor Portal Lite – P1)

4.1 Auth/Account (벤더 전용)

- 벤더 계정/역할
  - Vendor Admin: 회사 정보/사용자 관리, 카탈로그/가격표 업로드 권한
  - Vendor Member: RFQ 조회/회신, 일부 지표 조회
- 계정 생성/초대는 초기에는 **내부 운영자가 생성/초대 링크 발송**으로 제한
- Buyer Workspace와는 **데이터/도메인 분리**
  - 공통 Auth 인프라(예: 동일 IdP 또는 Auth 서버)는 쓸 수 있으나,
  - 벤더는 Buyer 조직/리스트/예산 등 내부 데이터에 직접 접근하지 않음

4.2 RFQ 리스트/상세

- RFQ 리스트
  - 컬럼: RFQ ID/제목, 생성일, 마감일, 상태(미회신/부분 회신/완료), 요청자(조직명)
  - 필터: 상태, 기간, 제품 카테고리(가능 시)
- RFQ 상세
  - 요청 제품별: 제품명/카탈로그번호(있으면)/필요 수량/요청 납기
  - Buyer가 작성한 특이사항/요청 사항(텍스트)

4.3 RFQ 회신 입력

- 웹 폼
  - 요청 제품별로 단가/납기/MOQ/대체품 필드 제공
  - 전체 할인율/메모 필드(선택)
- 엑셀 업로드 (최소 1개 방식 택1 가능)
  - 표준 템플릿 다운로드
  - 업로드 시 기본 검증(형식/필수 필드)
- 상태 전이
  - Draft(작성 중) → Submitted(제출 완료)
  - 부분 회신/전체 회신 여부는 Buyer 쪽 로직과 합의해 단순화

4.4 카탈로그/가격표 관리 (라이트)

- 업로드
  - xlsx/csv 포맷(내부 Vendor Data Kit 포맷과 최대한 호환)
- 검증/적용
  - zod 검증 결과 요약(오류 행 수, 필수 필드 누락 등)
  - diff 요약(신규/변경/삭제 건수)
  - 승인/적용(publish) 버튼
- 메타 정보
  - 최신 업데이트 날짜
  - Verified 배지 노출 조건(최근 X개월 내 업데이트 등은 추후 정의)

4.5 성과 지표 (라이트)

- 최소 지표
  - RFQ 수(기간 필터 기준)
  - 회신률(회신된 RFQ / 전체 RFQ)
  - 평균 회신 시간(요청 시각 ~ 회신 제출 시각)
- 확장 지표(후속)
  - 선정된 RFQ 수/비율
  - 기본 탈락 사유 카테고리(가격/납기/스펙/기타 등)

5. 데이터/아키텍처 원칙 (벤더 포털 관점)

- 데이터 분리
  - Buyer Workspace 데이터와 벤더 데이터는 테이블/스키마 레벨에서 분리
  - RFQ/제품/가격 정보만 “공통 도메인”으로 연결
- Vendor Data Kit 관계
  - P0.5~초기 P1: 내부 운영/데이터 담당자가 Vendor Data Kit 업로드/승인을 담당
  - 이후: 벤더 포털 라이트/Pro에서 벤더가 직접 자신의 카탈로그/가격표를 관리하는 방향으로 확장
- 지표 데이터 소스
  - 초기 P1: 내부 사용자가 입력/정리한 RFQ/회신 데이터를 기준으로 지표 산출
  - 벤더 포털 정착 후: 벤더 직접 회신/업로드 데이터까지 포함해 지표를 정교화

6. 비기능 요구사항 (벤더 포털에 특화)

- 보안
  - 벤더 계정은 자신 회사에 속한 RFQ/데이터만 접근 가능
  - 세션 타임아웃, 기본 2FA 옵션(후속)
- 사용성
  - RFQ 리스트/회신 플로우는 “3~4 클릭 내 제출”을 목표
  - 엑셀 업로드 실패 시, 어떤 행/열이 문제인지 명확한 에러 메시지 제공
- 성능
  - RFQ 리스트 첫 페이지 로딩 1~2초 이내 목표(벤더 기준)

7. KPI & 검증 (Vendor Portal Lite 관점)

- 핵심 KPI
  - 벤더 포털을 통해 회신된 RFQ 비율(이메일/기타 채널 대비)
  - 포털 사용 벤더의 평균 회신 시간 변화(도입 전 vs 후)
  - 벤더 포털 월간 활성 벤더 수(MAV)
- 정성 지표
  - 벤더 담당자 인터뷰: RFQ 처리 방식/시간/실수율 체감 변화

8. 로드맵 (벤더 측 관점 요약)

- P1 (Vendor Portal Lite)
  - 벤더 로그인/권한
  - RFQ 리스트/상세
  - RFQ 회신 입력(폼 또는 엑셀 중 최소 1개)
  - (옵션) 카탈로그/가격표 업로드 라이트
  - 기본 지표(현황/회신률/평균 회신 시간)
- P1.5~P2 (Vendor Pro 방향성)
  - 조직별 계약가/조건 관리
  - 고급 지표(라인업별 마진/선정률, 고객 세그먼트)
  - 워크플로우 연동(CRM/ERP/견적 시스템 등)
