## 3. 사용자 그룹 및 권한 통제 정책표

| 역할 | UserRole | 인원 | 허용 범위 | 차단 범위 |
|------|----------|------|----------|----------|
| 연구원 | RESEARCHER | 2~3명 | 제품 검색/비교, 견적 요청, 구매 등록, 재고 입출고, 점검, QR 스캔 | admin API, 제품 수정, 사용자 관리, 결제 |
| 구매 담당 | BUYER | 1명 | 구매 등록/관리, 재고 입고, 견적 조회 | admin API, 제품 수정, 사용자 관리 |
| 관리자 | ADMIN | 1명 | 전체 데이터 조회, 견적 승인/상태변경, 제품 정보 수정, 활동 로그 | 결제/빌링, 조직 삭제, 벤더 포털 |
| 소유자 | OWNER | 1명 (본인) | 조직 설정, 멤버 관리, 전체 관리자 기능 | 결제 (Stripe 미연동) |

## 4. 장애 대응 및 Triage 기준표

### 4-A. 심각도 분류 기준

| 등급 | 기준 | 대응 시간 | 대응 방식 | 예시 |
|------|------|----------|----------|------|
| **P0 (Blocker)** | 핵심 파이프라인 중단, 보안 침해, 데이터 유실 | **4시간 이내** | 즉시 핫픽스 배포. 해결 불가 시 베타 일시 중단. | 로그인 불가, 재고 수량 오반영, 권한 우회, DB 연결 끊김 |
| **P1 (Critical)** | 주요 기능 장애이나 우회 경로 존재 | **24시간 이내** | 다음 배포 사이클에 포함. 사용자에게 우회 방법 안내. | 견적 상태 변경 실패 (새로고침으로 해결), 사진 업로드 간헐 실패 |
| **P2 (Major)** | 불편하지만 업무 수행 가능 | **베타 기간 내** | 베타 종료 후 정식 출시 전 수정. | 필터 결과 정렬 오류, 한글 깨짐, UI 레이아웃 틀어짐 |
| **P3 (Minor)** | 미미한 문제, 개선 사항 | **정식 출시 전** | 백로그 등록. | 버튼 색상, 문구 오타, 비직관적 UX |

### 4-B. Triage 판정 플로우

```
이슈 접수
  │
  ├─ 보안/권한/인증 관련? ──── Yes ──→ P0 (즉시)
  │
  ├─ 데이터 유실/오반영? ──── Yes ──→ P0 (즉시)
  │
  ├─ 핵심 파이프라인 중단?
  │   (견적→구매→입고→재고)
  │   ├─ 중단, 우회 불가 ──────────→ P0
  │   └─ 중단, 우회 가능 ──────────→ P1
  │
  ├─ 주요 기능 장애? ──────── Yes ──→ P1
  │
  ├─ 불편 사항? ───────────── Yes ──→ P2
  │
  └─ 기타 ─────────────────────────→ P3
```

### 4-C. Kill Switch (베타 즉시 중단) 조건

아래 중 **하나라도** 해당 시 베타 즉시 중단:

- [ ] 인증/권한 우회로 비인가 데이터 접근 발생
- [ ] 재고 수량 무결성 파괴 (입출고 불일치)
- [ ] DB 연결 장애 30분 이상 지속
- [ ] Sentry에 동일 에러 50건/시간 이상 집중 발생
- [ ] 사용자 개인정보 노출 사고
